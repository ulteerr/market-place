# Чеклист private CSR

## Область применения

Используйте этот чеклист для приватных CSR-маршрутов:

- `/account/**`
- `/organizations/**`

Не используйте его для публичных SSR-маршрутов:

- `/`
- `/catalog/**`
- `/content/**`

## Когда запускать

- Перед финальной проверкой приватных кабинетов в PR.
- После изменений route rules для private-групп.
- После изменений bootstrap пользовательских настроек.
- После изменений инициализации темы, auth hydration или private layouts.

## Обязательные проверки

### 1. Маршрут остается CSR

- Убедитесь, что маршрут относится к private CSR policy.
- Не добавляйте в `/account/**` и `/organizations/**` зависимость от SSR-only поведения.

### 2. Для private-страниц нет SEO-требований индексации

- `h1`, `title`, `description` могут существовать для UX, но не являются gating-критерием индексации.
- Нельзя блокировать private-страницу из-за отсутствия public SEO-требований.

### 3. Для private-страниц не требуется schema.org

- `schema.org` не является обязательной для private routes.
- Не нужно внедрять public JSON-LD блоки в `/account/**` и `/organizations/**`.

### 4. Тема применяется до первого видимого кадра

- Перезагрузите страницу в `light` и `dark`.
- Убедитесь, что нет видимого мигания неправильной темы до завершения hydration.
- Первый видимый кадр уже должен быть в корректной теме.

### 5. Auth-зависимый UI стабилен на первом кадре

- Header, toolbar и layout shell не должны прыгать между анонимным и авторизованным состояниями после первого кадра.
- Настройки пользователя из auth/runtime не должны вызывать видимый layout flicker.

## Действия при падении проверки

- Считайте это регрессией private-shell.
- Проверьте bootstrap-логику в `nuxt.config.ts`, `useUserSettings` и private layouts.
- Не закрывайте задачу/PR, пока проблема первого кадра не устранена.
