# Frontend Documentation

This folder contains architecture and process docs for the Nuxt frontend (`frontend/`).

## Scope

- Current focus is admin area implementation.
- Public pages are present but documented only where they affect shared infrastructure.

## Sections

- `architecture.mdx`: app structure, runtime layers, and boundaries.
- `routing-and-layouts.mdx`: route model, middleware, and layout rules.
- `auth-and-session.mdx`: authentication flow and session persistence behavior.
- `ui-system.mdx`: UI component principles and Storybook usage.
- `data-and-api.mdx`: API integration model and conventions.
- `state-and-settings.mdx`: settings/state synchronization rules.
- `testing.mdx`: unit/e2e strategy and quality gates.
- `ssr-and-seo-checklist.mdx`: operational checklist for public SSR HTML verification.
- `private-csr-checklist.mdx`: operational checklist for private CSR shells and first-paint behavior.
- `performance-budget.mdx`: budget and pre-release checks for key public SSR routes.
- `public-qa-gate.mdx`: final gate before handing off public SSR pages to the next stage.
- `schema-registry.mdx`: single extension point and builder rules for public `schema.org`.
- `workflow.mdx`: day-to-day development workflow in Docker.

## Storybook

- URL in Docker: `http://localhost:6006`
- Run: `make front-storybook`
- Build: `make front-storybook-build`
- Story docs pages live in `frontend/docs/*.mdx`
- Visual regression scaffold: `.github/workflows/storybook-visual.yml` (requires `CHROMATIC_PROJECT_TOKEN`)

## Change policy

- Keep docs in this folder updated in the same PR as behavior changes.
- Update Storybook stories when component API or visual behavior changes.

# Документация Frontend

В этой папке собрана архитектурная и процессная документация Nuxt-фронтенда (`frontend/`).

## Область

- Текущий фокус разработки: админская часть.
- Публичные страницы документируются в той степени, в которой они затрагивают общую инфраструктуру.

## Разделы

- `architecture.mdx`: структура приложения, runtime-слои и границы.
- `routing-and-layouts.mdx`: маршрутизация, middleware и правила layout.
- `auth-and-session.mdx`: аутентификация и поведение сессии.
- `ui-system.mdx`: принципы UI-компонентов и работа со Storybook.
- `data-and-api.mdx`: модель интеграции с API и соглашения.
- `state-and-settings.mdx`: синхронизация состояния и настроек.
- `testing.mdx`: стратегия unit/e2e тестирования и quality gates.
- `ssr-and-seo-checklist.mdx`: операционный чеклист проверки public SSR HTML.
- `private-csr-checklist.mdx`: операционный чеклист private CSR shell и поведения первого кадра.
- `performance-budget.mdx`: budget и pre-release проверки для ключевых public SSR-маршрутов.
- `public-qa-gate.mdx`: финальный gate перед передачей public SSR-страниц на следующий этап.
- `schema-registry.mdx`: единая точка расширения и правила builder-слоя для public `schema.org`.
- `workflow.mdx`: ежедневный процесс разработки в Docker.

## Storybook

- URL в Docker: `http://localhost:6006`
- Запуск: `make front-storybook`
- Сборка: `make front-storybook-build`
- Документационные страницы: `frontend/docs/*.mdx`
- Каркас visual-regression: `.github/workflows/storybook-visual.yml` (нужен `CHROMATIC_PROJECT_TOKEN`)

## Политика изменений

- Обновляйте docs в этой папке в том же PR, где меняется поведение.
- Обновляйте stories в Storybook при изменении API или визуального поведения компонентов.
