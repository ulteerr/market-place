# Performance Budget

## Scope

This budget applies to key public SSR routes:

- `/`
- `/catalog/**`

Private routes are out of scope for this budget:

- `/account/**`
- `/organizations/**`

## Measurement mode

- Measure production build only.
- Use local or CI environment without dev server overhead.
- Re-run after `make cache-reset-all` when SSR artifacts may be stale.

## Target budget

For `/` and `/catalog/**`:

- HTML document size: up to `120 kB` before gzip
- Initial JS per route entry: up to `250 kB` gzip
- Initial CSS per route entry: up to `70 kB` gzip
- Largest client chunk on route: up to `320 kB` gzip

## Release rule

- Going above budget is allowed only with explicit justification in PR.
- Any regression on `/` or `/catalog/**` must be called out in review notes.
- Budget check is mandatory in pre-release review for public SSR changes.

Для `/` и `/catalog/**`:

- размер HTML-документа: до `120 kB` до gzip
- initial JS на route entry: до `250 kB` gzip
- initial CSS на route entry: до `70 kB` gzip
- largest client chunk на маршруте: до `320 kB` gzip

## Правило релиза

- Выход за budget допустим только с явным обоснованием в PR.
